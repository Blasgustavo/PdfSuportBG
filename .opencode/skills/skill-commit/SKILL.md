## What I do
- Validate commit messages follow format: `<type>(<scope>): <description>`
- Types: feat, fix, docs, style, refactor, test, chore
- Enforce 72 char max for description
- Use imperative mood, lowercase

## When to use me
Use this after every code change to ensure consistent commit messages.
